sizing

CSS is Awesome

by default, it's responsive all content will be visible, and the container will fit the available space width = 100%, height = min-content change the width, the height changes, still responsive, great change the height might be ok, might be bad you prefer things to look pretty, but the browser prefers everything to be visible so, overflow (the browser thinks you've made a mistake)
best practices - prefer not to set width and esp height - consider min/max-width/height (example of like a modal or something) - also min/max-content (demo) - scrollbars
signs normal flow might not be for you - i want all the elements to be the same size (consider grid) - i want elements to take up the available space (consider flex, or grid)

positioning

positioning - each element appears below the previous one, we can't change that - move vertically, using margin-top - move laterally using margin-left/margin-left with negative value/margin: auto
signs normal flow might not be for you - i want siblings to be positioned next to each other (consider flex, grid) - i want something centred vertically (consider flex, grid, transform) - i want something positioned relative to its siblings (consider flex, grid) if you're trying to do any of these things in normal flow, you're using the wrong tool and your life will be difficult
bonus sign that normal flow might not be for you: margin-collapse
