function getNames(users) {
	const names = [];

	for (let i = 0; i < user.length; i++) {
		const u = users[i];
		names.push(`${u.gName} ${u.famName}`);
	}

	return names;
}
					

function getNames(users) {
	return users.map((user) => `${user.gName} ${user.famName}`);
}
- imperative vs declarative loop in js - declarative is shorter because we're just telling the js engine what we want to happen, not how to do it - easier to understand, can mean less errors - trade-off: can't control how loop runs eg can't increment by 2
When a feature is available in a declarative language, not only is it easier to write, it's also more robust.

Positioning in CSS
HTML, in English, block level elements. There are 2 places you an put them. Of course, HTML also has inline elements, like images and text. They're placed next to each other, but how they work is kind of complicated, and not that important in this context, so let's ignore them.
CSS let's us make things prettier, and gives us a little bit of control over layouts - add space around elements via margins - move elements around with position and transform - change the element that the element is positioned relative

An element can be positioned

  • relative to an ancestor
